## 1. `core\core_error_handler.pix`

Скрипт обрабатывает ошибки при работе с транзакциями: логирует детали исключений, ведёт историю ошибок, отслеживает лимиты попыток, сохраняет скриншоты и пути к ним в глобальном контексте для последующей обработки и отчётности.

| Направление | Аргумент | Тип | Описание |
| :--- | :--- | :--- | :--- |
| **InOut** | `global_context` | Dictionary<System.String, System.Object> | Глобальный контекст выполнения приложения. Содержит пути, конфигурации и динамические параметры. |
| **In** | `transaction_handling_error` | System.Exception | Объект ошибки при транзакционной обработке. |
| **In** | `transactions_error_source` | System.String | Источник ошибки при транзакционной обработке. |
| **In** | `retry_counter` | System.Int32 | Счётчик текущих попыток. |
| **In** | `max_serial_errors` | System.Int32 | Максимальное число подряд идущих ошибок перед прерыванием процесса робота. |
| **In** | `errors_history` | List<System.String> | Список накопленных ошибок при обработке (N подряд). |
| **In** | `transactions_error_message` | System.String | Сообщение об ошибке при транзакционной обработке. |
| **In** | `screenshot_retry_counter_file` | System.String | Скриншот ошибки по текущей попытке. |

## 2. `core\core_get_transactions.pix`

Скрипт получает и формирует список транзакций из внешних источников, создаёт буферную таблицу, объединяет её с основной, добавляет технические столбцы, рассчитывает количество слотов для новых транзакций и контролирует дубли по составному первичному ключу с ведением журналов операций.

| Направление | Аргумент | Тип | Описание |
| :--- | :--- | :--- | :--- |
| **InOut** | `global_context` | Dictionary<System.String, System.Object> | Глобальный контекст выполнения приложения. Содержит пути, конфигурации и динамические параметры. |

## 3. `core\core_report_formatter.pix`

Скрипт формирует итоговый отчёт на основе обработанных транзакций: подготавливает человекочитаемую таблицу, преобразует статусы, генерирует HTML-отчёт, устанавливает итоговый статус выполнения и фиксирует время завершения.

| Направление | Аргумент | Тип | Описание |
| :--- | :--- | :--- | :--- |
| **InOut** | `global_context` | Dictionary<System.String, System.Object> | Глобальный контекст выполнения приложения. Содержит пути, конфигурации и динамические параметры. |

---

## 4. `core\core_workflow.pix`

Центральный координатор цикла работы с транзакциями: управляет обработкой и повторами, контролирует лимиты ошибок, обновляет статусы, подготавливает транзакции к бизнес-логике, обеспечивает устойчивость, восстановление после ошибки, уведомления и финализацию процесса с учётом технических окон и бизнес-исключений.

| Направление | Аргумент | Тип | Описание |
| :--- | :--- | :--- | :--- |
| **InOut** | `global_context` | Dictionary<System.String, System.Object> | Глобальный контекст выполнения приложения. Содержит пути, конфигурации и динамические параметры. |

---

## 5. `Main.pix`

Стартовый технический скрипт инициализирует рабочее окружение, проверяет технические условия запуска и обрабатывает уведомления и ошибки, обеспечивая корректное завершение процесса; дальнейшая бизнес-логика выполняется во внешних компонентах.

## 8. `process\process_transaction.pix`

Демонстрационный скрипт  
Скрипт выполняет бизнес-логику атомарной транзакции: поэтапно открывает карточку аккаунта, проверяет данные на соответствие бизнес-правилам и устанавливает финальный статус обработки `"completed"` в контексте транзакции, следуя общей архитектуре обработки транзакций.

| Направление | Аргумент | Тип | Описание |
| :--- | :--- | :--- | :--- |
| **InOut** | `global_context` | Dictionary<System.String, System.Object> | Глобальный контекст выполнения приложения. Содержит пути, конфигурации и динамические параметры. |
| **InOut** | `transaction_info` | Dictionary<System.String, System.Object> | Накопительный словарь по текущей транзакции. |
